---
title: "DAL Criteria API: query and filter entities"
description: "Build type-safe entity queries in Shopware using the Criteria class — filters, sorting, aggregations, associations, and pagination without raw SQL."
---

The `Criteria` class is Shopware's query builder for the Data Abstraction Layer (DAL). It replaces raw SQL and Doctrine's QueryBuilder entirely. You attach filters, sorting, aggregations, and associations to a `Criteria` instance, then pass it to an `EntityRepository` or send it as a JSON body to the Admin or Store API.

## Core methods

| Method | Description |
|---|---|
| `addFilter(Filter ...$filters)` | Append one or more filter conditions (ANDed together) |
| `addPostFilter(Filter ...$filters)` | Filters applied after aggregations (useful for faceted search) |
| `addSorting(FieldSorting ...$sorting)` | Append sort fields; multiple calls stack |
| `addAggregation(Aggregation ...$aggregations)` | Register aggregations by name |
| `addAssociation(string $path)` | Eager-load a relation; supports dot-notation (`categories.media`) |
| `addAssociations(array $paths)` | Shorthand for multiple `addAssociation` calls |
| `setLimit(?int $limit)` | Maximum number of results |
| `setOffset(?int $offset)` | Starting offset for pagination |
| `setIds(array $ids)` | Restrict results to a specific list of IDs |
| `setTotalCountMode(int $mode)` | `TOTAL_COUNT_MODE_NONE` (0), `TOTAL_COUNT_MODE_EXACT` (1), or `TOTAL_COUNT_MODE_NEXT_PAGES` (2) |
| `addQuery(ScoreQuery ...$queries)` | Boost results matching a condition without hard-filtering |

## Filter types

<AccordionGroup>
  <Accordion title="EqualsFilter">
    Exact match on a single field.

    ```php
    use Shopware\Core\Framework\DataAbstractionLayer\Search\Filter\EqualsFilter;

    $criteria->addFilter(new EqualsFilter('active', true));
    $criteria->addFilter(new EqualsFilter('manufacturer.name', 'Acme'));
    ```

    ```json
    { "filter": [{ "type": "equals", "field": "active", "value": true }] }
    ```
  </Accordion>

  <Accordion title="EqualsAnyFilter">
    Matches when the field value is in a list (SQL `IN`).

    ```php
    use Shopware\Core\Framework\DataAbstractionLayer\Search\Filter\EqualsAnyFilter;

    $criteria->addFilter(new EqualsAnyFilter('categoryIds', ['abc123', 'def456']));
    ```

    ```json
    { "filter": [{ "type": "equalsAny", "field": "categoryIds", "value": ["abc123", "def456"] }] }
    ```
  </Accordion>

  <Accordion title="ContainsFilter">
    Case-insensitive substring match (`LIKE %value%`).

    ```php
    use Shopware\Core\Framework\DataAbstractionLayer\Search\Filter\ContainsFilter;

    $criteria->addFilter(new ContainsFilter('name', 'shirt'));
    ```

    ```json
    { "filter": [{ "type": "contains", "field": "name", "value": "shirt" }] }
    ```
  </Accordion>

  <Accordion title="PrefixFilter / SuffixFilter">
    Anchored substring match from the start or end of the value.

    ```php
    use Shopware\Core\Framework\DataAbstractionLayer\Search\Filter\PrefixFilter;
    use Shopware\Core\Framework\DataAbstractionLayer\Search\Filter\SuffixFilter;

    $criteria->addFilter(new PrefixFilter('productNumber', 'SW-'));
    $criteria->addFilter(new SuffixFilter('name', '-XL'));
    ```
  </Accordion>

  <Accordion title="RangeFilter">
    Numeric or date range. Operators: `gte`, `gt`, `lte`, `lt`.

    ```php
    use Shopware\Core\Framework\DataAbstractionLayer\Search\Filter\RangeFilter;

    $criteria->addFilter(new RangeFilter('stock', [
        RangeFilter::GTE => 1,
        RangeFilter::LTE => 100,
    ]));
    ```

    ```json
    {
      "filter": [{
        "type": "range",
        "field": "stock",
        "parameters": { "gte": 1, "lte": 100 }
      }]
    }
    ```
  </Accordion>

  <Accordion title="MultiFilter (AND / OR)">
    Combine filters with `AND` or `OR` logic.

    ```php
    use Shopware\Core\Framework\DataAbstractionLayer\Search\Filter\MultiFilter;
    use Shopware\Core\Framework\DataAbstractionLayer\Search\Filter\EqualsFilter;
    use Shopware\Core\Framework\DataAbstractionLayer\Search\Filter\RangeFilter;

    $criteria->addFilter(new MultiFilter(MultiFilter::CONNECTION_OR, [
        new EqualsFilter('active', true),
        new RangeFilter('stock', [RangeFilter::GT => 0]),
    ]));
    ```

    ```json
    {
      "filter": [{
        "type": "multi",
        "operator": "or",
        "queries": [
          { "type": "equals", "field": "active", "value": true },
          { "type": "range", "field": "stock", "parameters": { "gt": 0 } }
        ]
      }]
    }
    ```
  </Accordion>

  <Accordion title="NotFilter / NandFilter / NorFilter / XOrFilter">
    Logical negation and set operators. `NotFilter` wraps a list of filters and negates them (NOT ANY).

    ```php
    use Shopware\Core\Framework\DataAbstractionLayer\Search\Filter\NotFilter;
    use Shopware\Core\Framework\DataAbstractionLayer\Search\Filter\EqualsFilter;
    use Shopware\Core\Framework\DataAbstractionLayer\Search\Filter\MultiFilter;

    $criteria->addFilter(new NotFilter(MultiFilter::CONNECTION_AND, [
        new EqualsFilter('active', false),
    ]));
    ```
  </Accordion>

  <Accordion title="NotEqualsFilter / NotEqualsAnyFilter">
    Convenience negation wrappers for exact and list matches.

    ```php
    use Shopware\Core\Framework\DataAbstractionLayer\Search\Filter\NotEqualsFilter;
    use Shopware\Core\Framework\DataAbstractionLayer\Search\Filter\NotEqualsAnyFilter;

    $criteria->addFilter(new NotEqualsFilter('taxId', null));
    $criteria->addFilter(new NotEqualsAnyFilter('status', ['cancelled', 'failed']));
    ```
  </Accordion>
</AccordionGroup>

## Sorting

```php
use Shopware\Core\Framework\DataAbstractionLayer\Search\Sorting\FieldSorting;

$criteria->addSorting(
    new FieldSorting('price', FieldSorting::ASCENDING),
    new FieldSorting('name', FieldSorting::DESCENDING),
);
```

In an API request body:

```json
{
  "sort": [
    { "field": "price", "order": "ASC", "naturalSorting": false },
    { "field": "name", "order": "DESC" }
  ]
}
```

Set `naturalSorting: true` to sort strings like `"item-2"` before `"item-10"`.

## Aggregations

```php
use Shopware\Core\Framework\DataAbstractionLayer\Search\Aggregation\Metric\SumAggregation;
use Shopware\Core\Framework\DataAbstractionLayer\Search\Aggregation\Bucket\TermsAggregation;

$criteria->addAggregation(new SumAggregation('total-stock', 'stock'));
$criteria->addAggregation(new TermsAggregation('by-manufacturer', 'manufacturer.name'));
```

```json
{
  "aggregations": [
    { "name": "total-stock", "type": "sum", "field": "stock" },
    { "name": "by-manufacturer", "type": "terms", "field": "manufacturer.name" }
  ]
}
```

## Pagination

```php
$criteria->setLimit(25);
$criteria->setOffset(50); // page 3
$criteria->setTotalCountMode(Criteria::TOTAL_COUNT_MODE_EXACT);
```

<Note>
  `TOTAL_COUNT_MODE_NONE` (default) skips the `COUNT(*)` query entirely — use it whenever you don't need a page count.
  `TOTAL_COUNT_MODE_NEXT_PAGES` fetches `limit * 5 + 1` rows and is faster than an exact count for "has next page" UIs.
</Note>

## Associations

```php
// Eager-load a flat association
$criteria->addAssociation('manufacturer');

// Dot-notation for deep relations
$criteria->addAssociation('categories.media.thumbnails');

// Apply filters to the association itself
$criteria->getAssociation('prices')
    ->addFilter(new EqualsFilter('quantityStart', 1))
    ->addSorting(new FieldSorting('quantityStart'));
```

```json
{
  "associations": {
    "manufacturer": {},
    "categories": {
      "associations": {
        "media": {}
      }
    }
  }
}
```

## Repository usage (PHP)

<CodeGroup>
```php Search with filters
use Shopware\Core\Framework\DataAbstractionLayer\Search\Criteria;
use Shopware\Core\Framework\DataAbstractionLayer\Search\Filter\EqualsFilter;
use Shopware\Core\Framework\DataAbstractionLayer\Search\Sorting\FieldSorting;
use Shopware\Core\Framework\DataAbstractionLayer\Search\Aggregation\Metric\SumAggregation;

$criteria = new Criteria();
$criteria->addFilter(new EqualsFilter('active', true));
$criteria->addSorting(new FieldSorting('name', FieldSorting::ASCENDING));
$criteria->addAggregation(new SumAggregation('total-stock', 'stock'));
$criteria->setLimit(10);
$criteria->setTotalCountMode(Criteria::TOTAL_COUNT_MODE_EXACT);

$result = $this->productRepository->search($criteria, $context);

$products  = $result->getEntities();  // EntityCollection
$totalCount = $result->getTotal();
$stockSum   = $result->getAggregations()->get('total-stock');
```

```php Fetch by IDs
$criteria = new Criteria(['id-1', 'id-2', 'id-3']);
$criteria->addAssociation('cover.media');

$result = $this->productRepository->search($criteria, $context);
```
</CodeGroup>

## Admin API request body

```json
POST /api/search/product

{
  "limit": 5,
  "page": 1,
  "filter": [
    { "type": "equals", "field": "active", "value": true },
    { "type": "range",  "field": "stock",  "parameters": { "gt": 0 } }
  ],
  "sort": [
    { "field": "name", "order": "ASC" }
  ],
  "associations": {
    "manufacturer": {},
    "cover": { "associations": { "media": {} } }
  },
  "aggregations": [
    { "name": "by-manufacturer", "type": "terms", "field": "manufacturer.name" }
  ],
  "total-count-mode": 1
}
```

## Store API request body

```json
POST /store-api/product

{
  "limit": 24,
  "filter": [
    { "type": "equalsAny", "field": "categoryIds", "value": ["<category-id>"] }
  ],
  "sort": [{ "field": "price", "order": "ASC" }],
  "includes": {
    "product": ["id", "name", "price", "cover"]
  }
}
```

<Note>
  Use the `includes` map to restrict which fields are serialized in the response. This can significantly reduce payload size in Store API responses.
</Note>
